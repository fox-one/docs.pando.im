授权

大多数API是公开访问的，但有些仅供认证用户使用。

Sign /me with keystores​

Usually, you can use the a regular Mixin authentication token for GET https://api.mixin.one/me with scope "FULL".

The token should be generated by your own Mixn Application. Please visit Developer Dashboard to create an application, and visit Mixin Developers Docs for the signing procedure.

POST /oauth​

In merely cases, for example, you want to provide user a customized swap GUI, you can use the OAuth to get a token for the user.

Call this API instead of Mixin Messenger's OAuth API to exchange the access token with code.

The code must be issued by 4swap.

API 终点 URL

https://api.4swap.org/api/oauth

认证和选项

授权公开访问
限制No limitation

负载

{ // 来自 Mixin Messenger's OAuth 的 code "code":       "28fefbf1284d90ceb10bddd517fab2a716f4713ebe3f3299a9fd4d881b4c8b54", // leave "broker_id" and "label" to empty if you don't want to use other brokers. "broker_id":  "", "label":      "" }

请求示例

curl -X POST "https://api.4swap.org/api/oauth --data PAYLOAD" -H "Content-Type: application/json"
Response
{
  "ts": 1627697766645,
  "data": {
    "scope": "PROFILE:READ ASSETS:READ",
    // OAuth 的 token
    "token": "..."
  }
}

The token is compatible with Mixin API. It's fine to use it to get information from Mixin API.
